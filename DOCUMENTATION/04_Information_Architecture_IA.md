# الهيكلية المعلوماتية (Information Architecture)
## Information Architecture - منصة قيمّ

---

## 📋 نظرة عامة

هذه الوثيقة تحدد الهيكلية الكاملة لمنصة قيمّ، بما في ذلك:
- التنظيم الهرمي للمعلومات
- هيكل التنقل الرئيسي لكل دور
- Sitemap شامل
- نظام الـ Labels والتسميات
- هيكل المحتوى التعليمي

---

## 🏗️ المبادئ الأساسية للـ IA

### 1. الوضوح (Clarity)
- تسميات واضحة ومفهومة باللغة العربية
- تجنب المصطلحات التقنية المعقدة
- استخدام أيقونات مساعدة

### 2. الاتساق (Consistency)
- نفس التسميات في جميع الصفحات
- نفس موقع عناصر التنقل
- أنماط تفاعل موحدة

### 3. الكفاءة (Efficiency)
- الوصول للمحتوى المهم بأقل نقرات (3 نقرات كحد أقصى)
- إمكانية العودة السريعة
- Breadcrumbs واضحة

### 4. التوسع (Scalability)
- قابلية إضافة أقسام جديدة
- دعم عدد كبير من المستخدمين والمحتوى
- هيكل مرن للتعديل المستقبلي

## 🌐 Sitemap الرئيسي

```
منصة قيمّ
│
├─── 🌍 الصفحات العامة (Public)
│    ├─── الصفحة الرئيسية (/)
│    ├─── من نحن (/about)
│    ├─── الخدمات (/services)
│    ├─── تواصل معنا (/contact)
│    ├─── الصفحات المخصصة (/pages/{slug})
│    └─── التسجيل الذاتي
│         ├─── معلم (/register/teacher/{token})
│         ├─── طالب (/register/student/{token})
│         └─── ولي أمر (/register/parent/{token})
│
├─── 🔐 المصادقة (Authentication)
│    ├─── تسجيل الدخول (/login)
│    ├─── التحقق بخطوتين (/two-factor/verify)
│    ├─── نسيت كلمة المرور (/forgot-password)
│    ├─── إعادة تعيين كلمة المرور (/reset-password/{token})
│    └─── تغيير كلمة المرور (/password/change)
│
├─── 👑 Super Admin (/admin/*)
│    ├─── [التفاصيل في القسم التالي]
│
├─── 🏫 School Admin (/school-admin/*)
│    ├─── [التفاصيل في القسم التالي]
│
├─── 👩‍🏫 Teacher (/teacher/*)

│
├─── 👦 Student (/student/*)

```

```
│    │    │    │    ├─── Images

│    │    │         └─── Activity 3
│    │    │

│    ├─── Meaning 2

```

### مثال عملي: قيمة "الأمانة"

```
🌟 الأمانة (Value)
│
├─── 📖 مفهوم الأمانة (Concept)
│    │
│    ├─── 💡 حفظ الودائع (Meaning)
│    │    │
│    │    ├─── 📚 درس: ما هي الأمانة؟ (Lesson)
│    │    │    ├─── نص توضيحي
│    │    │    ├─── قصة قصيرة
│    │    │    ├─── صورة توضيحية
│    │    │    │
│    │    │    └─── 📝 الأنشطة
│    │    │         ├─── نشاط 1: اكتب موقفاً عن الأمانة
│    │    │         ├─── نشاط 2: رسم معبر
│    │    │         └─── نشاط 3: اختبار قصير
│    │    │
│    │    └─── 📚 درس: نماذج من الأمانة
│    │         └─── الأنشطة...
│    │
│    ├─── 💡 الصدق في القول (Meaning)
│    └─── 💡 أداء الواجبات (Meaning)
│
├─── 📖 تطبيقات الأمانة (Concept)
└─── 📖 ثمرات الأمانة (Concept)
```

### قواعد التنقل في المحتوى

1. **التسلسل الإجباري:**
   - لا يمكن فتح القيمة التالية حتى إكمال الحالية 100%
   - الدروس تُفتح بالتسلسل داخل كل معنى
   - الأنشطة يجب إنجازها لإكمال الدرس

2. **حساب التقدم:**
   ```
   تقدم القيمة = (عدد الدروس المكتملة / إجمالي الدروس) × 100
   الدرس المكتمل = جميع أنشطته مُقيّمة ومقبولة
   ```

3. **الحالات الممكنة:**
   - 🔒 **مقفل:** لم يصل الطالب بعد
   - 📖 **قيد التنفيذ:** بدأ لكن لم ينته
   - ✅ **مكتمل:** أنهى جميع الدروس والأنشطة

## 🔄 تدفقات التنقل الرئيسية

### Flow 1: من تسجيل الدخول إلى Dashboard

```
/login
  ├─ تسجيل الدخول بنجاح
  ├─ 2FA (إذا كان مفعلاً)
  └─ إعادة التوجيه حسب الدور:
      ├─ Super Admin → /admin/dashboard
      ├─ School Admin → /school-admin/dashboard
      ├─ Teacher → /teacher/dashboard
      ├─ Student → /student/dashboard
      └─ Parent → /parent/dashboard
```

### Flow 2: Student - من Dashboard إلى إنجاز نشاط

```
/student/dashboard
  └─ خريطة التعلم
      └─ /student/learning-path
          └─ اختيار قيمة
              └─ اختيار مفهوم
                  └─ اختيار معنى
                      └─ اختيار درس
                          └─ قراءة المحتوى
                              └─ اختيار نشاط
                                  └─ /student/activities/{id}
                                      └─ كتابة الإجابة
                                          └─ تسليم
                                              └─ رسالة نجاح + عودة للـ Dashboard
```

### Flow 3: Teacher - مراجعة نشاط

```
/teacher/dashboard
  └─ الأنشطة المعلقة (15)
      └─ /teacher/pending-activities
          └─ اختيار نشاط
              └─ /teacher/activities/{id}/review
                  └─ قراءة الإجابة
                      └─ إعطاء درجة + ملاحظات
                          └─ حفظ
                              └─ النشاط التالي (أو العودة للقائمة)
```

### Flow 4: School Admin - قبول طالب

```
/school-admin/dashboard
  └─ طلبات التسجيل (3 جديدة)
      └─ /school-admin/registration-requests
          └─ اختيار طلب
              └─ /school-admin/registration-requests/{id}
                  └─ مراجعة البيانات
                      └─ قبول + تعيين الفصل
                          └─ تأكيد
                              └─ رسالة نجاح + البريد يُرسل تلقائياً
```

## 📏 قواعد الـ IA

### Rule 1: 3-Click Rule
- أي معلومة مهمة يجب الوصول إليها في 3 نقرات كحد أقصى
- الأنشطة المعلقة: Dashboard → نقرة واحدة ✅
- تسليم نشاط: Dashboard → خريطة التعلم → نشاط → تسليم = 3 نقرات ✅

### Rule 2: Breadcrumbs
- يجب عرض Breadcrumbs في جميع الصفحات (ما عدا Dashboard)
- مثال: `لوحة التحكم > المحتوى > القيم > تعديل`

### Rule 3: العودة للخلف
- زر "رجوع" واضح في جميع الصفحات الفرعية
- زر "إلغاء" في جميع النماذج

### Rule 4: الحالة النشطة
- تمييز واضح للصفحة الحالية في القائمة الجانبية
- تمييز العناصر النشطة (Active State)

### Rule 5: الإشعارات
- عرض عداد الإشعارات في القائمة
- مثال: `الأنشطة المعلقة (🔴 15)`
